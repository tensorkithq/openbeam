# Contributing

**Prompts and plans over code.**

## Got a bug or request?

Open an issue as a **prompt** — a clear instruction an AI agent can act on.

## Want to make a change?

Submit a **plan PR**, not a code PR. A plan is a markdown spec with:

- **Goal** — what it accomplishes
- **Files** — paths to create, modify, or delete
- **Spec** — step-by-step instructions detailed enough for Claude or Codex to one-shot it
- **Verification** — how to confirm it works

Once the plan is approved, implementation follows from it.

Typos and one-line fixes are fine as direct code PRs.
